Raw PDF text sounds bad out loud.

PDF extraction gives characters. Speech needs clean text.

A2A cleanup runs between pypdf and chunk_texts. It fixes text, not meaning.

Cut page numbers, headers, footers, hyphen breaks, orphan lines, and table debris. Log every cut.

Fix page junk

Strip page furniture, rebuild words split across lines, and patch paragraph breaks.

Preserve order

Keep the author's order. Put every deletion in the cleanup report.

Prepare speech

Shorten giant sentences, normalize headings, and escape characters that trip SSML.

A2A insertion point

PDF -> pypdf raw text
    -> local A2A cleanup agent
    -> cleaned voice-ready text
    -> chunk_texts
    -> hosted or local TTS
    -> finalized MP3
Chunk files and .done markers make failed TTS runs easy to resume. Broken batch jobs should resume cleanly.

The voice box can move.

Pick the voice last: local when it sounds good, hosted when language coverage or quality wins.

Keep the files boring

Keep parsing, cleanup, chunking, validation, synthesis, and finalization separate.

Improve cleanup first

Bad text makes every voice sound bad. The cleanup report is where you catch the damage.

Swap voices later

Fast, listenable local TTS can replace hosted TTS later.
